CSS Utilities

Breakpoints

scss/helpers/_breakpoints.scss

This SASS utility is an attempt to replicate the functionality of the Foundation Breakpoint Mixin and operates in much the same manner. It consists of three parts: a map of predefined $breakpoints and $containers breakpoints, along with breakpoint() and container() mixins.

Warning: These mixins cannot be deployed in a Modern Project style -- its SASS compiler is out-of-date and cannot the functions necessary to make this work. For now the recommendation is to continue to use foundation but to remove all imports save for the global styles.

Warning: To deploy these mixins in the Component Library Project style you will need to update the list of excluded at-rules on line 15 of _config/.stylelintrc.json to include use, else, and container.

The predefined $breakpoints are:

$breakpoints: ( xsmall: 0, small: 400px, medium: 768px, large: 1024px, xlarge: 1280px, xxlarge: 1440px, xxxlarge: 1920px } $containers: ( xsmall: 0, small: 360px, medium: 720px, large: 960px, xlarge: 1152px, xxlarge: 1296px, xxxlarge: 1728px );

These breakpoints may be altered or added to as needed; the only requirement is that they should be listed in ascending size. (Certain parameters for the mixins require them to access the next largest breakpoint.) Note that the container breakpoints have been adjusted down from the media breakpoings, mostly to account for the global margns we add to our layouts.

The breakpoint() and container() mixins are simple:

@include breakpoint($size, $direction) { ... } @include container($size, $direction) { ... }

The mixins must be called with a mandatory $size parameter, which can be any of the defined values in the $breakpoints map, or an arbitrary string. In either case, the output will be:

@media screen and (min-width: $size) { ... } @container and (width > $size) { ... }

If the $size parameter can be resolved to one of the defined values in the $breakpoints map, the mixin may be called with an optional $direction parameter which modifies the output:

until: @media screen and (max-width: [$size - 1px]) { ... } until: @containter (width < size - 1px) { ... } down: @media screen and (max-width: [next largest $size - 1px]) { ... } down: @container (width < $size) { ... } only: @media screen and (min-width: $size) and (max-width: [next largest $size - 1px]) { ... } only: @container and (width >= $size) and (width < [next largest $size]) { ... } up: @media screen and (min-width: $size) { ... } up: @container and (width >= $size) { ... }

Please note that the output will be broken if you use a direction other than 'up' with the ultimate $breakpoint.

Easings

scss/helpers/_easings.scss

This SASS utility converts the predefined cubic-bezier curves from easings.net and turns them into SASS variables. The variable names are identical to the system names. That's it.

Visibility

scss/helpers/_breakpoints.scss
scss/helpers/_visibility.scss

This SASS utility is an attempt to replicate the functionality of the Foundation Visibility Classes and operates in much the same manner. It is dependent on the breakpoints utility as it relies on its map of predefined $breakpoints.

Warning: These mixins cannot be deployed in a Modern Project style -- its SASS compiler is out-of-date and cannot the functions necessary to make this work. For now the recommendation is to continue to use foundation but to remove all imports save the global styles and visibility classes.

Warning: To deploy these mixins in the Component Library Project style you will need to update the list of excluded at-rules on line 15 of _config/.stylelintrc.json to include use, else, and container.

When compiled, the utility adds the following classes to the stylesheet:

  • show-for-sr which visually hides an element without toggling it to display: none or removing it from the document tree
  • show-on-focus which visually hides an element without toggling it to display: none or removing it from the document tree, and reveals it only when the element or one of its children receives focus

It adds the following classes for each $breakpoints with a width greater than 0:

  • show-for-[$size] which applies display: none to an element at all sizes below the indicated $breakpoint
  • hide-for-[$size] which applies display: none at the indicated $breakpoint
  • show-for-c-[$size] and hide-for-c-[$size] which use container queries instead of media queries

Javascript Utilities

Rocket Tabs

ejs/plugins/br-tabs.js
scss/plugins/_br-tabs.scss

This is an extremely minimal section tab script intended to get you the most bang for your buck and to avoid the hassles of dealing with Foundation Tabs. It implements UI best practices outlined by Adrian Roselli and Heydon Pickering.

  • Structure
  • Interactions
  • Options
  • Methods

The most minimal version of this component looks like this:

<div class="br-tabs"> <ul class="br-tabs__navigation"> <li class="br-tabs__item"><a href="#tab-panel-1" class="br-tabs__link" id="tab-1">First Tab</a></li> </ul> <div class="br-tabs__content"> <div class="br-tabs__panel" id="tab-panel-1" aria-labelledby="tab-1"> <p>Content</p> </div> </div> </div> <script> window.addEventListener('load', (event) => { document.querySelectorAll('.br-tabs').forEach( function( element ) { element.tab = Object.assign(new BRTabs(element)); }); }); </script>

The top-level br-tab does not require a unique ID but should probably have one. The component should not be nested.

The br-tabs__navigation, br-tabs__item, br-tabs__link and br-tabs__panel items have appropriate aria-hidden aria-selected, role and tabindex values applied after the component is activated.

The br-tabs__link should anchor to a unique ID on the corresponding br-tabs__panel and should have a unique ID of its own.

br-tabs__panel element should have a unique ID and an aria-labelledby attribute pointing back to the appropriate br-tabs__link.

These elements may be swapped out for other element types (such as <section>) as needed, but it is important to preserve the hierarchy and class names.

Activating a link via click displays the panel with the corresponding ID.

Currently inactive tabs are not keyboard focusable.

Using the up, down, left and right arrow keys while focused on a link navigates you to the next link in the sequence and displays the corresponding panel.

Deep linking can be achieved by adding a hash to the URL; the panel with the corresponding ID will be opened when the component initialized.

  • open (default: 0): Indicates which panel to open when initially activated. May be an ineger or string. If an integer, open the panel whose index corresponds to that integer; if a string, open the panel whose ID corresponds to that string.
  • [node].tab.destroy(): Destructures the entire tab component.
  • [node].tab.open(panel): May take an integer or string. If an integer, open the panel whose index corresponds to that integer; if a string, open a panel the panel whose ID corresponds to that string.

Rocket Toggle

ejs/plugins/br-toggle.js
scss/plugins/_br-toggle.scss

This is an extremely minimal section toggling script intended to get you the most bang for your buck and to avoid the hassles of dealing with Foundation Accordions. It implements UI best practices outlined by Adrian Roselli and Heydon Pickering; and uses a grid-based auto height transition trick as explained by Keith J. Grant.

The most minimal version of this component looks like this:

<div class="br-toggle"> <button class="br-toggle__button" id="toggle-1" disabled>Button</button> <div class="br-toggle__panel" aria-labelledby="toggle-1" > <div class="br-toggle__content"> <p>Content</p> </div> </div> </div> <script> window.addEventListener('load', (event) => { document.querySelectorAll('.br-toggle').forEach( function( element ) { element.toggle = Object.assign(new BRToggle(element, { options })); }); }); </script>

The top-level br-toggle does not require a unique ID but should probably have one. The component may be nested (but probably shouldn't be).

The br-toggle__button should be disabled by default (and you should include styling to make sure it looks correct in those instances. The disabled attribute is removed when the component is activated, and aria-label and aria-expanded attributes are added.

The br-toggle__panel element should be aria-labelledby the appropriate br-toggle__button. Attributes of role="region" and aria-hidden="true" are added when the component is activated.

These elements may be swapped out for other element types (such as <section>) as needed, with the exception of the br-toggle__button element.

Please note that the default styling for this is excessively minimal to allow for maximum flexibility.

  • Activating the button via click, enter, or space bar toggles the state of the next adjacent section. If accordion is true, any other open sections are closed.
  • If navigation is set, buttons are also activated when the user mouses over them, and all panels are closed when the user mouses out of the toggle.
  • Using the up and down arrow keys while focused on a button navigates you to the next button in the current toggle (but not into or out of nested toggles).
  • Deep linking can be achieved by adding a hash to the URL; the panel associated with the button at the corresponding ID will be opened when the component initializes.

  • accordion (default: true): Indicates whether this is an accordion (where only one item may be open at a time) or a group of individually collapsible sections. At desktop resolutions (1024px+) the first item in an accordion is opened by default unless this is overruled by other options.
  • navigation (default: null): If set to an integer, uses that as a minimum screen size to trigger hover-based navigation (mouseenter/mouseleave).
  • open (default: 0): Indicates which panel to open when initially activated. May be a boolean, integer, or string. If false do not open anything; if an integer, open the panel whose index corresponds to that integer; if a string, open a the panel whose ID corresponds to that string (# not required).

  • [node].toggle.closeAll(): Toggle all panels closed.
  • [node].toggle.destroy(): Destructures the entire toggle component.
  • [node].toggle.open(panel): May take an integer or string. If an integer, open the panel whose index corresponds to that integer; if a string, open a panel the panel whose ID corresponds to that string.
